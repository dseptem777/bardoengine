## 1. Complete Templates

Each command's `tag` field becomes the full template with all params and sensible defaults.

**Before:** `/minigame_qte` â†’ `#MINIGAME: type=qte`
**After:** `/minigame_qte` â†’ `#MINIGAME: type=qte, key=SPACE, timeout=1.5, autostart=true`

The writer selects the value they want to change â€” everything else works out of the box. All params documented inline via the template itself.

Applies to all commands that have parameters: minigames, VFX, horror system, boss fights, audio, etc.

`filterCommands(query, config)` generates dynamic sub-commands from the project's `story-config.json`.

**Implementation:**

- BardoEditor passes `projectConfig` through node data so PassageNode can forward it to `filterCommands`

## 3. Conditional Blocks

New commands in `Logic` category for Ink's inline conditional syntax.

| Command | Template |
|---------|----------|
| `/if` | `{variable == value: text}` |
| `/ifelse` | `{variable == value: text \| other text}` |
| `/ifelseif` | `{variable == value: text \| variable2 == value2: text2 \| fallback text}` |

**Contextual autocomplete applies:**
```
/if â†’ shows variables from config + system vars:
  /if:hp              â†’ {hp > 0: text}
  /if:has_key         â†’ {has_key == true: text}
  /if:minigame_result â†’ {minigame_result == 1: text}
```

**Smart defaults per variable type:**
- Number stats â†’ `> 0`
- Boolean / inventory â†’ `== true`
- System result vars (`minigame_result`, `willpower_passed`, etc.) â†’ `== 1`

**Complex conditionals** are pure Ink syntax â€” the writer extends the template manually:
- AND: `{hp > 0 and has_key == true: text}`
- OR: `{hp > 0 or courage > 5: text}`
- Nested: `{hp > 0: {has_key: door opens | locked} | you're dead}`

The `/if` description explains AND/OR/nesting so writers learn organically.

## 4. Improved Descriptions

Every command description teaches what it does, not just labels it.

**Format:**
1. What it does (plain language)
2. Key params or behavior
3. For result-producing commands: variable name and values
4. Max ~120 chars

**Examples:**
```
/shake       â†’ "Shakes the screen. Intensity: light, normal, heavy"
/music       â†’ "Plays background music. Loops until stopped or changed"
/music_stop  â†’ "Fades out current music"
/minigame_qte â†’ "Quick-time event â€” player presses key before timeout. Result: minigame_result (1=win, 0=lose)"
/if          â†’ "Conditional text: {var == val: text}. Complex: 'and'/'or' for multiple conditions"
/willpower_start â†’ "Starts willpower drain. Player clicks to resist. Speed: slow, normal, fast"
/boss_start  â†’ "Starts boss fight. Follow with BOSS_CHECK to test result"
```

**New category color:** Logic = blue (joins existing: VFX=purple, Minigame=green, Horror=red, Combat=orange, Game=cyan).
