## 11. Tooltips & Inline Guides

**Goal:** Context-sensitive help so writers never feel lost.

**Files to modify:**
- `src/editor/components/Tooltip.jsx` — **NEW** — reusable tooltip component
- `src/editor/utils/helpTexts.js` — **NEW** — all help text strings
- Various components — add `<Tooltip>` wrappers to buttons, fields, panels

**Tooltip types:**

### Hover tooltips
- Every toolbar button: brief description + shortcut key
- Tag badges on nodes: "This tag does X. Click to learn more."
- Choice condition icon: "This choice only appears when the condition is met"
- Hub/knot type badges: "Hub: a location players return to. Knot: a one-time passage."

### First-use guides (show once, dismiss with "Got it")
- First time opening editor: "Welcome! Double-click the canvas to create your first passage."
- First time editing a node: "Type your story text here. Use / to insert effects and actions."
- First time adding a choice: "Choices let your reader decide what happens next. Connect them to other passages."
- First time opening preview: "This plays your story as players will see it. Watch for errors!"

### Info panels (? icon)
- Variables panel: "Variables store information that changes during the story (health points, items found, etc.)"
- Config panel each tab: brief explanation of what stats/inventory/achievements do
- Chapter field: "Chapters help organize your story. Nodes with the same chapter name are grouped together."

**Implementation:**
- Tooltips: CSS-only with `data-tooltip` attribute + `:hover::after` pseudo-element (no library needed)
- First-use guides: Track shown state in localStorage, render as dismissible banner
- Info panels: Small `(?)` icon that toggles an info box

**Implementation steps:**
1. Create `Tooltip.jsx` — wrapper component with hover-triggered popup
2. Create `helpTexts.js` with all tooltip/guide strings (easy to update/translate)
3. Add hover tooltips to all toolbar buttons in BardoEditor
4. Add first-use guide banners with localStorage tracking
5. Add `(?)` icons to panel headers

## 12. Search & Replace in Content

**Goal:** Find and replace text across all nodes (e.g., rename a character).

**Files to modify:**
- `src/editor/components/SearchReplace.jsx` — **NEW**
- `src/editor/BardoEditor.jsx` — Add Ctrl+H shortcut, manage panel state

**UI:**
```
┌─ Search & Replace ──────────────────────────┐
│ Find:    [old character name          ] 🔍   │
│ Replace: [new character name          ]      │
│                                              │
│ [☐ Case sensitive] [☐ Tags only]            │
│                                              │
│ Found in 7 nodes:                            │
│ ┌──────────────────────────────────────────┐ │
│ │ 📻 intro — "...old character name..."    │ │
│ │ 📻 scene_2 — "...old character name..."  │ │
│ │ 📻 scene_5 — "...old character name..."  │ │

│ └──────────────────────────────────────────┘ │

│ [Replace All (7)] [Replace Selected] [Close] │
└──────────────────────────────────────────────┘
```

**Behavior:**
- Searches all node content (text + tags)
- Shows matching nodes with context snippet
- Click on result jumps to node (like Ctrl+P search)
- "Tags only" checkbox limits search to tag lines
- Replace updates content in all matching nodes
- Undo-able (single undo step for bulk replace)

**Implementation steps:**

2. Search function: iterate all nodes, regex match on `data.content`
3. Replace function: update all matching nodes' content, push single undo snapshot
4. Add Ctrl+H shortcut in BardoEditor

- `src/editor/hooks/useUndoRedo.js` — Expose history array + labels

**UI:**
```
┌─ History ──────────────┐
│                        │
│ ● Added node "intro"   │  ← current
│ ○ Connected intro→ch1  │
│ ○ Edited "chapter_1"   │
│ ○ Added choice          │
│ ○ Deleted "old_node"   │
│ ○ Added stat action     │
│ ○ Project created       │

```

**Behavior:**

- Current position highlighted
- Click to jump to any point (undo/redo multiple steps)
- Labels auto-generated by diffing snapshots:

  - Content changed → "Edited node X"
  - Edge added/removed → "Connected/Disconnected X → Y"
  - Config changed → "Updated settings"

**Implementation steps:**

2. Add `getHistoryLabels()` to expose timeline data
3. Add `jumpToSnapshot(index)` function
4. Create `HistoryPanel.jsx` — timeline list with click-to-jump
5. Add toggle button to toolbar
6. Auto-generate labels by diffing consecutive snapshots (node count diff, edge count diff, content length diff)

## 14. Enhanced Validation

**Goal:** Catch common writer mistakes before they hit compile errors.

**Files to modify:**
- `src/editor/utils/generateInk.js` — Expand `validateGraph()` with more checks
- `src/editor/components/ValidationPanel.jsx` — **NEW** (or enhance existing warnings display)
- `src/editor/BardoEditor.jsx` — Show validation panel

**New validations:**

| Check | Severity | Message |
|-------|----------|---------|
| Empty node content | Warning | "Node 'X' has no text — readers will see a blank screen" |
| Dead-end without END | Warning | "Node 'X' has no outgoing connections — story will stop here" |
| Unreachable nodes | Warning | "Node 'X' can't be reached from the start" |
| Choice without target | Error | "Choice 'Y' in node 'X' doesn't lead anywhere" |
| Referenced variable not defined | Error | "Condition uses 'hp' but this variable isn't defined" |
| Stats/achievements in tags but not in config | Warning | "Tag references stat 'hp' but it's not configured in Settings" |
| Duplicate choice text | Warning | "Node 'X' has two identical choices" |
| Very long passage (>500 words) | Info | "Node 'X' is very long — consider splitting into multiple passages" |
| No entry point | Error | "No 'start' node found — story won't begin" |
| Circular path without choice | Warning | "Nodes X → Y → X create a loop without player choice" |

**UI enhancement:**
```
┌─ Validation ──────────────────────────────────┐
│ ❌ 2 Errors  ⚠️ 3 Warnings  ℹ️ 1 Info        │
│                                                │
│ ❌ Choice "Open door" in node "hallway"       │
│    doesn't lead anywhere                       │
│    [Jump to node]                              │
│                                                │
│ ⚠️ Node "secret_room" can't be reached        │
│    from the start                              │
│    [Jump to node]                              │
│                                                │
│ ℹ️ Node "long_exposition" is very long         │
│    (523 words) — consider splitting            │
│    [Jump to node]                              │
└────────────────────────────────────────────────┘
```

**Behavior:**
- Runs automatically on export/preview
- Can be triggered manually (toolbar button)
- Click "Jump to node" centers canvas on that node
- Errors block export with confirmation ("Export anyway?")
- Warnings shown but don't block

**Implementation steps:**
1. Add new validation checks to `validateGraph()` in generateInk.js
2. Add variable reference checking (parse conditions + tag params for variable names, cross-check with variables list)
3. Add config cross-reference checking (stat IDs in tags vs config definitions)
4. Create `ValidationPanel.jsx` with severity filtering, jump-to-node
5. Add toolbar button + auto-run on export/preview
6. Write tests for each validation rule
